## 6. Adding a Peer (local, out-of-band)

There is no server-mediated friend request. Each side adds the other from their two
public keys, obtained out-of-band.

```mermaid
sequenceDiagram
    actor Alice
    actor Bob
    participant OOB as Out-of-band

    Alice->>OOB: Share (x25519_A, ed25519_A)
    OOB->>Bob: Bob receives Alice's keys
    Bob->>Bob: initialize_peer(...) -> peers/hex(id_A)/peer.pub

    Bob->>OOB: Share (x25519_B, ed25519_B)
    OOB->>Alice: Alice receives Bob's keys
    Alice->>Alice: initialize_peer(...) -> peers/hex(id_B)/peer.pub

    Note over Alice,Bob: id = sha256(x25519 ‖ ed25519)<br/>keys never pass through the server
```

## 7. P2P Connection Flow

```mermaid
sequenceDiagram
    participant Alice
    participant STUN as STUN (UDP)
    participant Server
    participant Bob

    Note over Alice,Bob: Discover public UDP mapping, then announce it

    Alice->>STUN: signed StunPing (persistent UDP socket)
    STUN->>Alice: observed public ip:port
    Alice->>Server: Announce(token, alice_udp)
    Bob->>Server: Announce(token, bob_udp)
    Server->>Alice: PeerOnline(bob_id, bob_udp)

    Note over Alice,Bob: Relay brokers the punch

    Alice->>Server: RequestHolePunch(token)
    Server->>Bob: RequestHolePunch(alice_id)
    Bob->>Server: RequestAccepted(token)
    Server-->>Alice: PunchHole(bob_udp, ts)
    Server-->>Bob: PunchHole(alice_udp, ts)

    Note over Alice,Bob: Direct: signed ephemeral handshake over UDP

    Alice->>Bob: Ping(ts) x N (hole punch)
    Bob->>Alice: Ping(ts) x N
    Alice->>Bob: PeerMessage::Init(ephemeral_pub_A), Ed25519-signed
    Bob->>Alice: PeerMessage::Init(ephemeral_pub_B), Ed25519-signed
    Note over Alice,Bob: X25519(ephemeral) -> session_key<br/>per-direction subkeys, monotonic counters

    Note over Alice,Bob: Fallback: punch times out -> TURN relay

    Alice->>Server: RelayData(bob_id, encrypted PeerMessage)
    Server->>Bob: RelayData(alice_id, encrypted PeerMessage)
    Note over Alice,Bob: same crypto, static-DH session key,<br/>server only moves ciphertext
```

## 9. Filesystem Layout

```mermaid
flowchart TB
    Root["~/.sechat/"]
    Key["identity.key<br/>encrypted privkeys<br/>124B"]
    Pub["identity.pub<br/>both pubkeys<br/>64B"]
    Server["server<br/>relay address (plaintext)"]
    Aliases["aliases<br/>identity_hash -> name"]
    Peers["peers/"]

    Peer1["hex(id_A)/"]
    Peer2["hex(id_B)/"]
    Peer1Pub["peer.pub<br/>x25519 + ed25519"]
    Peer1Chat["chat.db<br/>sled encrypted"]
    Peer2Pub["peer.pub"]
    Peer2Chat["chat.db"]

    Root --> Key
    Root --> Pub
    Root --> Server
    Root --> Aliases
    Root --> Peers
    Peers --> Peer1
    Peers --> Peer2
    Peer1 --> Peer1Pub
    Peer1 --> Peer1Chat
    Peer2 --> Peer2Pub
    Peer2 --> Peer2Chat

    classDef enc fill:#f8d7da,stroke:#721c24
    classDef pub fill:#e1f5e1,stroke:#2d6a2d
    classDef dir fill:#cfe2ff,stroke:#0d6efd

    class Key,Peer1Chat,Peer2Chat enc
    class Pub,Peer1Pub,Peer2Pub pub
    class Root,Peers,Peer1,Peer2 dir
```

> `id = sha256(x25519_pub ‖ ed25519_verifying)` is the peer identity used for
> directory names, presence routing, and mailbox addressing. `server` and
> `aliases` are local config; aliases are never sent to the relay.
